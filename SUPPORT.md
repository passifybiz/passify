# Support

Thanks for using Passify. Here is the fastest way to get help.

## Documentation first

- **Guides and API reference:** https://passify.biz/docs
- **OpenAPI specification:** [`openapi.yaml`](./openapi.yaml) or
  https://passify.biz/api/openapi
- **SDK usage:** [`packages/sdk/README.md`](./packages/sdk/README.md)

## Questions and discussion

- **GitHub Discussions:** for questions, ideas, and integration help.
- **Bug reports:** open an issue using the **Bug report** template.
- **Feature requests:** open an issue using the **Feature request** template.

Please search existing issues and discussions before opening a new one.

## What belongs where

| You want to…                          | Use                                  |
| ------------------------------------- | ------------------------------------ |
| Report reproducible broken behavior   | GitHub issue (Bug report)            |
| Propose a capability or change        | GitHub issue (Feature request)       |
| Ask "how do I…" / get integration help| GitHub Discussions                   |
| Report a security vulnerability       | See [`SECURITY.md`](./SECURITY.md)   |
| Account, billing, or commercial topics| support@passify.biz                  |

## Response expectations

This is maintained alongside active product development. We triage issues and
discussions regularly but cannot guarantee a response time for community
channels. Commercial support terms are available on request via
support@passify.biz.

Please do not use public channels for security issues — follow
[`SECURITY.md`](./SECURITY.md) instead.
